## Design Decisions

- **Decision:** Adapt atomic-facts pattern for decisions, not entities
  **Context:** Considered porting the full 3-tier memory (life/areas/) from main workspace
  **Rationale:** Forge tracks engineering decisions and chaos findings, not people/companies. Different domain needs different entities.
  **Human input:** Asked whether to add full 3-tier; accepted the adapted recommendation

- **Decision:** Security Auditor is informational/non-blocking, not a pipeline gate
  **Context:** Inspired by garrytan's CSO ("informational only, please hire a pentester")
  **Rationale:** Blocking on security findings would slow the pipeline for false positives. HIGH/CRITICAL findings are escalated to the human via parent session notification instead.
  **Human input:** Directed implementation based on gstack CSO tweet

- **Decision:** Security Auditor uses 8/10 confidence gate
  **Context:** gstack CSO uses 8/10 for daily, 2/10 for comprehensive
  **Rationale:** Zero noise > zero misses. Users stop reading noisy reports. Forge's auditor runs per-workflow so daily mode confidence is appropriate.

- **Decision:** Testing strategy uses 5 tiers with escalating cost
  **Context:** Forge has zero tests currently
  **Rationale:** Tier 0+1 (free, structural) catch 80% of regressions. Higher tiers add agent and pipeline validation at increasing cost. Matches gstack's approach (static → E2E → LLM-as-judge) adapted for Forge's multi-agent architecture.
