## Rules Added
- `docs/rules/supply-chain-security.md` — Pin all 3rd-party GitHub Actions to commit SHAs
  - Reference SHA table for all actions in use
  - Rationale: mutable tags are a supply chain attack vector
- `docs/rules/pr-workflow.md` (PR #4) — All changes via PRs from feature branches
  - Branch naming convention
  - No direct pushes to main
  - Auto-merge (squash) + draft PR rules (PR #5)

## Lessons Learned
- Auto-merge without branch protection merges PRs without review
  - Auto-merge workflow was added and immediately auto-merged itself + revert PRs
  - Reverted from all repos; will re-add only after branch protection is configured
